## 一、异步编程类问题

### 1. `Promise.all` vs `Promise.allSettled` 区别

#### 🔍 问题分类

- **技术领域**：JavaScript 异步编程
- **考察重点**：Promise 高级用法、实际场景应用能力
- **面试阶段**：初级/中级前端必问题（出现概率 > 80%）

#### 💡 面试官真实意图

| 问题             | 面试官想确认                   |
| :--------------- | :----------------------------- |
| "说说区别"       | 你是否真正理解而不仅是背文档   |
| "什么时候用哪个" | 你是否有工程思维和实际项目经验 |
| "返回结构是什么" | 你是否关注技术细节             |

#### 🚀 高分回答模板（30秒版）

```
【一句话定义】
"Promise.all 要求'全员到齐'，1人迟到全队解散； 
Promise.allSettled 是'点名签到'，迟到早退都记录。"

【项目实例】
"在用户批量操作项目中：
- 用 allSettled 处理100个用户请求 → 成功的直接更新，失败的自动收集ID重试
- 如果用 all，1个用户报错会导致99个成功操作全回滚，用户体验很糟"

【选择原则】
"强依赖任务（如支付+发券+通知）→ 用 all（必须全成功）
 弱依赖任务（如仪表盘数据聚合）→ 用 allSettled（要独立处理结果）"
```

#### ❓ 高频追问及应对策略

| 面试官可能问                         | 完美回答技巧                                                 | 避坑指南                                                |
| :----------------------------------- | :----------------------------------------------------------- | :------------------------------------------------------ |
| "返回结构具体什么样？"               | "固定格式对象： ✅ 成功：`{status: 'fulfilled', value: '结果'}` ❌ 失败：`{status: 'rejected', reason: '错误'}` 全部按顺序塞进数组" | ✘ 不要说"返回数组"（太笼统） ✓ 强调"按顺序"和"固定字段" |
| "为什么不用 catch 处理 all 的失败？" | "catch 只知道'挂了'，但不知道'谁挂了'。 allSettled 直接告诉你第3个请求超时，能精准修复问题" | ✘ 不要只说"allSettled 更好" ✓ 对比说明实际价值          |
| "allSettled 兼容性如何？"            | "IE不支持，但现代项目都用 polyfill 或 Babel 转换。 我们团队规范要求：需要容错的场景必须用 allSettled" | ✘ 不要说"不知道" ✓ 展示工程实践经验                     |

## 二、字符串操作类问题

### 1. `substring` vs `substr` 区别

#### 🔍 问题分类

- **技术领域**：JavaScript 基础语法
- **考察重点**：基础 API 掌握程度、现代开发规范意识
- **面试阶段**：初级前端必问题（出现概率 > 70%）

#### 💡 面试官真实意图

| 问题                 | 面试官想确认               |
| :------------------- | :------------------------- |
| "说说区别"           | 你是否死记硬背还是真正理解 |
| "实际开发用哪个"     | 你是否关注行业最佳实践     |
| "为什么有这两个方法" | 你是否有技术演进意识       |

#### 🚀 高分回答模板（20秒版）

```
【核心区别】
"substring 看'落刀位置'：第二个参数是结束位置（不吃这个点）
 substr 看'切几段'：第二个参数是截取长度"

【香肠例子】
"str = 'hello world'
 substring(0,5) → 从0切到5（不吃5）→ 'hello'
 substr(0,5) → 从0切5段 → 'hello'"

【行业现状】
"但实际开发中我们**不用 substr**！
 因为它在IE有兼容问题，现代项目都用 slice：
 - slice 和 substring 行为一致
 - 还支持负数索引（str.slice(-3)取最后3字符）"
```

## 三、面试应答心法

### ✅ 通用回答公式

```
【1句话定义】 + 【生活化比喻】 + 【项目实例】 + 【选择原则】
```

> 例：
> *"Promise.all 要求全员到齐（1句话定义），像团队必须全员到岗才能开工（比喻）。
> 我在用户批量操作中用 allSettled 处理100个请求（实例），
> 因为强依赖任务用 all，弱依赖用 allSettled（原则）。"*

### 🛡️ 遇到不会的问题怎么办？

1. **不慌张**：说"这个问题我项目中用得少，但根据原则..."
2. **转话题**：关联到你熟悉的领域（例："这让我想起在XX项目中处理类似问题..."）
3. **表态度**：加一句"入职后我会重点学习这个场景"

### 📈 从回答中展现的3个关键素质

| 你展示的素质 | 面试官看到的价值 | 实现方式               |
| :----------- | :--------------- | :--------------------- |
| **技术深度** | 能解决复杂问题   | 用具体例子说明技术选型 |
| **工程思维** | 能产出高质量代码 | 强调团队规范和最佳实践 |
| **学习能力** | 能快速适应项目   | 提及"入职后会重点学习" |

## 附：面试前10分钟速记卡

### 字符串方法速查

```
substring = 刀起(0)刀落(5) → 吃[0]到[4] → 现代开发用 slice
substr = 刀起(0)切5段 → 吃[0][1][2][3][4] → 已淘汰！
```

> 2️⃣ **展现思考过程**（为什么这么选）
> 3️⃣ **绑定实际场景**（证明不是背文档）
>
> **记住：面试官更想招一个"能解决问题的人"，而不是"会背文档的机器人"**
